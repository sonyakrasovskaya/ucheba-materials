# Безопасная загрузка папки «Учеба» в приватный GitHub-репозиторий (macOS)

---

## 1. Краткий план

1. Проверить Git и установить/авторизовать GitHub CLI (`gh`).
2. Убедиться, что в папке есть актуальный `.gitignore` (мусор и большие файлы не попадут в репозиторий).
3. Найти файлы больше ~90 MB (GitHub не принимает файлы **>100 MB**).
4. Инициализировать Git, сделать **dry-run** (посмотреть список файлов без добавления).
5. Добавить файлы, сделать первый коммит.
6. Создать **приватный** репозиторий через `gh` и выполнить первый push.
7. Проверить результат.

**Лимит GitHub:** один файл не больше **100 MB**. Репозиторий лучше держать в разумных размерах (гигабайты — не лучший вариант). Большие файлы лучше исключать через `.gitignore` или использовать Git LFS (отдельная настройка).

## 2. Шаги с командами

### Шаг 0. Перейти в папку

```bash
cd "/Users/sofya/Desktop/Учеба"
```

Дальше все команды выполняются из этой папки (если не указано иное).

---

### Шаг 1. Проверка Git

```bash
git --version
```

Должно вывести что-то вроде: `git version 2.39.5`. Если команда не найдена — установи Xcode Command Line Tools: `xcode-select --install`.

---

### Шаг 2. Установка и авторизация GitHub CLI

**Установка через Homebrew:**

```bash
brew install gh
```

Если Homebrew нет: установи с [brew.sh](https://brew.sh).

**Вход в GitHub (откроется браузер):**

```bash
gh auth login
```

Выбери: **GitHub.com** → **HTTPS** → **Login with a web browser**, введи код из терминала на странице GitHub.

**Проверка:**

```bash
gh auth status
```

Должно быть: `Logged in to github.com as <твой_логин>`.

---

### Шаг 3. Проверка больших файлов (>90 MB)

Эти файлы **не должны** попасть в репозиторий (лимит GitHub 100 MB). Выполни:

```bash
find "/Users/sofya/Desktop/Учеба" -type f -size +90M 2>/dev/null
```

Если список не пустой — убедись, что такие файлы или папки указаны в `.gitignore`. В твоём `.gitignore` уже добавлены: чекпоинты NLP, большие модели (`.safetensors`, `optimizer.pt`, `best_model.pt`), `YearPredictionMSD.txt`, `*.mp4`, `*.rar`. Если появились новые большие файлы — добавь их в `.gitignore` (или маску, например `*.zip` для больших архивов).

Файл `.gitignore` уже лежит в `/Users/sofya/Desktop/Учеба`. Он исключает:

### Шаг 5. Инициализация Git (если ещё не инициализирован)

```bash
cd "/Users/sofya/Desktop/Учеба"
git init
```

Если увидишь `Reinitialized existing Git repository` — репозиторий уже был создан, можно идти дальше.

---

### Шаг 6. Безопасный dry-run перед `git add .`

**Смысл:** посмотреть, **какие файлы будут добавлены**, без реального добавления. Так можно убедиться, что нет лишнего и нет файлов больше 100 MB.

Сначала общий статус:

```bash
git status
```

Потом «симуляция» добавления (ничего не добавляется, только список):

```bash
git add -n .
```

Просмотри вывод. Все перечисленные пути попадут в первый коммит. Если видишь лишнее (личные файлы, большие данные) — добавь их в `.gitignore` и снова выполни `git add -n .`, пока список не будет устраивать.

---

### Шаг 7. Реальное добавление файлов и первый коммит

Когда dry-run показал нормальный набор файлов:

```bash
git add .
git status
```

Ещё раз посмотри список. Затем первый коммит:

```bash
git commit -m "Первая загрузка: учебные материалы"
```

---

### Шаг 8. Создание приватного репозитория и push

**Создать приватный репозиторий** и связать его с текущей папкой (remote `origin`):

```bash
gh repo create ucheba-materials --private --source=. --remote=origin --description "Учебные материалы"
```

Имя репозитория можно заменить на другое (например, `study-materials`). После этой команды репозиторий уже создан на GitHub и привязан к папке.

**Переименовать ветку в `main` (если нужно) и отправить код:**

```bash
git branch -M main
git push -u origin main
```

Если `gh repo create` уже инициализировал ветку `main`, просто:

```bash
git push -u origin main
```

При необходимости `gh` попросит подтвердить создание репозитория — ответь Yes.

---

### Шаг 9. Проверка

- В браузере открой: `https://github.com/<твой_логин>/ucheba-materials` — должны быть папки и файлы, репозиторий помечен как Private.
- В терминале:

```bash
gh repo view ucheba-materials
```

## 3. Ошибки и решения

| Ошибка | Что сделать |
|--------|--------------|
| `gh: command not found` | Выполни `brew install gh`, затем `gh auth login`. |
| `authentication required` или 403 при `git push` | Выполни `gh auth login` и снова `git push origin main`. |
| `failed to push some refs` / `rejected` | На GitHub уже есть коммиты (например, README). Выполни: `git pull origin main --rebase`, затем `git push origin main`. |
| `File XXX is 125.00 MB; this exceeds GitHub's file size limit` | Файл XXX не игнорируется. Добавь его (или маску) в `.gitignore`, затем: `git rm --cached путь/к/XXX`, сделай коммит и снова `git push`. |
| В dry-run вижу лишние файлы | Добавь их пути или маски в `.gitignore`. Выполни `git reset HEAD`, затем снова `git add -n .` и проверь список. |
| Хочу залить файл больше 100 MB | Либо не заливать (оставить в `.gitignore`), либо настроить [Git LFS](https://git-lfs.github.com/) и добавить нужный тип файлов: `git lfs track "*.psd"` и т.п. |

## 4. Как обновлять репозиторий потом

После изменений в папке «Учеба» (новые файлы, правки в конспектах и т.д.):

```bash
cd "/Users/sofya/Desktop/Учеба"
git status
git add .
git commit -m "Краткое описание изменений"
git push origin main
```

Перед коммитом можно посмотреть изменения: `git status` и `git diff`.
